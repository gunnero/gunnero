# Security policy

## Supported version

The current version of this profile repository is supported. It contains public Markdown content and validation automation, not a deployed application.

## Reporting a vulnerability

Please report security concerns privately by emailing [aleksandar.dimovski@me.com](mailto:aleksandar.dimovski@me.com). Do not open a public issue for a suspected vulnerability.

Include the affected file or workflow, a clear description, reproduction details when applicable, potential impact, and any suggested mitigation. An acknowledgement will be sent after the report has been reviewed; no fixed response time is promised.

Please keep reports confidential until a resolution is agreed. Do not include credentials, customer information, private URLs, server topology, or other sensitive production data. Vulnerabilities concerning a linked project should be reported using that repository's own security policy.
